# 阶段暂停倒计时 — 需求文档

## 背景

当前 awf-run 在关键决策点（PLAN 完成后、DESIGN 结束后、COMMIT 前、FINISH 后）会暂停等待用户确认。标准模式下这是合理的，但用户希望在不需要逐项确认时自动推进，同时保留随时介入的能力。

## 核心方案

在阶段切换暂停点增加**倒计时自动继续**机制：

```
阶段 A 完成
    │
    ▼
┌─────────────────────────┐
│ ⏳ 将在 N 秒后自动进入阶段 B  │
│ 回复任意内容即暂停工作流     │
└─────────────────────────┘
    │
    ├── 倒计时内用户回复 → 暂停，记录断点，等待 /awf-run --resume
    │
    └── 倒计时结束无回复 → 自动进入阶段 B
```

## 功能范围

- **包含**：
  - 暂停点倒计时显示 + 自动继续
  - 用户随时回复打断暂停
  - 打断后自动保存断点，事后 `/awf-run --resume` 继续
  - 可配置倒计时时长和适用范围
  - `--auto` 模式直接跳过倒计时（不显示，秒进）

- **不包含**：
  - 实时动态倒计时 UI（CLI 限制，用 ScheduleWakeup 模拟）
  - 打断后自动回到原处（需要用户主动 resume）

## 适用暂停点

| 暂停点 | 默认倒计时 |
|--------|----------|
| PLAN → CODE/DESIGN | 15s |
| DESIGN → CODE | 10s |
| COMMIT 前 | 10s |
| FINISH → 下一里程碑 | 15s |

## 用户配置

```json
{
  "pauseCountdown": {
    "enabled": true,
    "duration": 15
  }
}
```

后续可扩展每个暂停点的独立时长配置。
